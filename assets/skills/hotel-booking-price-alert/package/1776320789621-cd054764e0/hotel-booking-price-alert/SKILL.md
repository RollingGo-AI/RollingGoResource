# 酒店价格异常波动监控与预警（Price Alert Advisor）

> 多时间点采集价格 → 计算波动率 → 识别异常 → 输出预警建议。

## 场景说明

用户需要在特定目的地或针对特定酒店，持续观察价格走势，识别暴涨/暴跌信号，
并在节假日、大型活动、出行旺季等关键节点前后获得操作建议（立即预订 / 观望 / 规避）。

适用场景：
- **差旅预算管控**：出差目的地房价超出公司标准，需要找到合规时间窗口预订
- **目的地盯盘**：旅游旺季前持续观察，等待价格回落
- **竞品监控**：OTA 运营人员追踪竞争对手酒店的动态定价策略
- **节假日波动观察**：春节、五一、国庆等节点前后的价格异动分析

---

## 触发示例

1. "五一假期上海酒店价格涨了多少？现在订还来得及吗？"
2. "帮我盯一下三亚的酒店，看看最近价格有没有异常波动。"
3. "我们公司差旅标准是 500 元/晚，北京现在的酒店价格超标了吗？"
4. "国庆前后成都酒店价格怎么变化的？什么时候订最划算？"

---

## 工作流程

### Step 1：收集监控参数

**必须从用户处获取：**
- 目的地城市或具体酒店名称
- 监控的日期范围（入住日期 + 住宿晚数）

**可从上下文推断或使用默认值：**
- 预警阈值：默认 ±20%（价格相对基准涨跌超过此幅度触发预警）
- 对比基准：默认使用"当前搜索均价"作为基准
- 星级范围：默认不限（可按用户差旅标准设定）
- 预算上限：用户提及时记录，用于超标预警

**询问话术示例：**
> "我来帮你分析价格波动。请问是哪个城市、哪段日期？有没有预算上限或星级要求？"

---

### Step 2：获取当前价格基准

运行搜索获取目标日期的当前市场价格，作为波动分析的基准：

```bash
npx --yes --package rollinggo@latest rollinggo search-hotels \
  --origin-query "<目的地>酒店价格" \
  --place "<目的地>" \
  --place-type city \
  --country-code CN \
  --check-in-date YYYY-MM-DD \
  --stay-nights N \
  --adult-count 2 \
  --size 10 \
  --format table
```

从结果中提取：
- 各酒店当前报价
- 计算**市场均价**（所有结果的平均值）
- 记录**最低价**和**最高价**

---

### Step 3：多时间窗口对比采集

为识别波动趋势，对同一目的地采集**相邻日期**的价格，构建价格序列：

**采集策略（根据用户需求选择）：**

| 场景 | 采集窗口 | 说明 |
|------|----------|------|
| 节假日前后 | 节前 3 天、节中、节后 3 天 | 各搜索一次，对比涨跌幅 |
| 差旅预算管控 | 目标日期 ±7 天 | 找出预算内的可预订窗口 |
| 竞品监控 | 固定日期，每次对话更新 | 与上次记录对比变化量 |
| 旺季盯盘 | 每周采集，追踪趋势 | 输出价格走势描述 |

**示例：节假日前后对比搜索**

```bash
# 节前（五一前 3 天）
npx --yes --package rollinggo@latest rollinggo search-hotels \
  --origin-query "上海五一前酒店" \
  --place "上海" --place-type city --country-code CN \
  --check-in-date 2026-04-28 --stay-nights 1 \
  --adult-count 2 --size 10 --format table

# 节中（五一当天）
npx --yes --package rollinggo@latest rollinggo search-hotels \
  --origin-query "上海五一酒店" \
  --place "上海" --place-type city --country-code CN \
  --check-in-date 2026-05-01 --stay-nights 1 \
  --adult-count 2 --size 10 --format table

# 节后（五一后 3 天）
npx --yes --package rollinggo@latest rollinggo search-hotels \
  --origin-query "上海五一后酒店" \
  --place "上海" --place-type city --country-code CN \
  --check-in-date 2026-05-05 --stay-nights 1 \
  --adult-count 2 --size 10 --format table
```

---

### Step 4：异常波动计算与预警判定

**波动率计算公式：**

```
波动率 = (当前价格 - 基准价格) / 基准价格 × 100%
```

**预警等级判定：**

| 预警等级 | 触发条件 | 含义 |
|----------|----------|------|
| 🔴 高预警 | 波动率 > +40% 或 < -30% | 价格严重异常，需立即决策 |
| 🟡 中预警 | 波动率 +20%~+40% 或 -15%~-30% | 价格明显偏离，建议关注 |
| 🟢 低预警 / 正常 | 波动率在 ±20% 以内 | 价格正常波动范围 |

**差旅超标预警（有预算上限时额外判断）：**
- 当前均价 > 预算上限 → 触发超标预警，同时给出预算内可选日期

---

### Step 5：输出预警报告

**标准输出格式：**

```
📍 目的地：[城市]  |  📅 监控日期：[日期范围]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

价格快照
  基准均价：¥XXX/晚
  当前均价：¥XXX/晚
  最低价：¥XXX（[酒店名]）
  最高价：¥XXX（[酒店名]）

波动分析
  整体波动率：+XX%  [预警等级]
  异常酒店：[酒店名] 涨幅 +XX%（¥XXX → ¥XXX）

操作建议
  [根据预警等级给出具体建议]

价格走势（多时间点时）
  节前：¥XXX  →  节中：¥XXX  →  节后：¥XXX
  趋势：[上涨中 / 已见顶回落 / 持续低位]
```

**操作建议话术模板：**

| 情况 | 建议话术 |
|------|----------|
| 🔴 高预警（暴涨） | "价格已严重偏高，若行程可调整建议错峰；若日期固定，**立即预订**锁定当前价格，后续可能继续上涨。" |
| 🟡 中预警（上涨） | "价格处于上涨通道，建议**近期预订**，避免进一步涨价。" |
| 🟡 中预警（下跌） | "价格仍在下行，可**短暂观望** 2-3 天，但需关注是否触底反弹。" |
| 🔴 高预警（暴跌） | "价格异常低位，可能是促销或临时降价，建议**立即锁定**，此类低价通常持续时间短。" |
| 🟢 正常 | "价格处于正常范围，可按计划预订，无需特别操作。" |
| 差旅超标 | "当前均价 ¥XXX 超出差旅标准 ¥XXX。建议选择 [日期] 入住，届时均价约 ¥XXX，符合预算。" |

## 本场景专属规则

1. **必须给出操作建议**：仅输出数字没有价值，每次分析必须附带明确的"立即预订 / 观望 / 规避"建议。
2. **波动率基于同类酒店均价**：不得用单一最低价作为基准，避免因个别特价酒店导致误判。
3. **节假日场景必须覆盖节前/节中/节后三个时间点**：单点价格无法判断趋势。
4. **差旅超标时必须给出合规替代方案**：不得仅告知超标，需同时提供预算内的可选日期或酒店。
5. **竞品监控需标注价格变化方向**：用 ↑↓ 符号标注，便于快速识别。

---

## 无结果时的处理

1. 扩大搜索范围（`--distance-in-meter` 增大或切换到城市级别）
2. 增加候选数量（`--size` 提升到 15-20）
3. 放宽星级限制
4. 若仍无结果，告知用户该目的地/日期暂无可用数据，建议更换日期重试
